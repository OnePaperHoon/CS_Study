# 바이너리 서치 (Binary Search)

## 주요 특징 및 원리

- 정렬 필수: 반드시 데이터가 오름차순 또는 내림차순으로 정렬되어야 합니다.
- 탐색 범위 축소 : 매 단계마다 탐색 범위를 절반으로 줄여나가기 때문에 속도가 매우 빠릅니다.

- 동작 과정:
	1. 정렬된 리스트의 중앙값과 찾으려는 값을 비교합니다.
	2. 찾으려는 값이 중앙값보다 크면, 중앙값의 오른쪽 부분 리스트를 대상으로 다시 탐색 합니다.
	3. 찾으려는 값이 중앙값보다 작으면, 중앙값의 왼쪽 부분 리스트를 대상으로 다시 탐색 합니다.
	4. 이 과정을 반복하여 값을 찾거나, 더 이상 탐색할 범위가 없으면 탐색을 종료합니다.


### 장점
- 빠른 속도: 탐색 범위가 절반씩 줄어들기 때문에, 대규모 데이터에서도 빠르게 값을 찾을 수 있습니다.
- 낮은 시간 복잡도: 시간 복잡도가 O(log n)으로, 매우 효율적입니다.
