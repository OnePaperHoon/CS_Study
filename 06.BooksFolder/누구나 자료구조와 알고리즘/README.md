# Big O

1. O(1): 상수 시간 알고리즘에서의 표기 \
N의 수와 상관 없이 3단계를 가지는 알고리즘을 표기 할때 O(3)이라 한다고 생각이 들겠지만 \
O(1)으로 표기하는게 맞다.

2. O(100): 빅 오의 본질 더 파고들기\
데이터의 크기와 상관없이 100단계가 걸리는 상수 시간 알고리즘이 있을때 \
O(N)인 알고리즘보다 다소 성낭이 우수하다고 볼것인가의 질문.\
N이 100이 되기 전까지는 O(100)의 성능이 안좋다고 볼수 있지만 \
데이터 크기가 커지면 O(N)은 무한대의 단계 까지 갈수 있기 때문에\
O(N)이 전반적으로 덜 효율적이라 할 수 있다.
